What I use.

A list of the hardware, software, and infrastructure I actually work with day to day — not aspirational gear, just the kit that's open on my screen right now. Inspired by uses.tech.

Hardware & desk

Laptop
Personal Windows laptop (16 GB RAM) — the workhorse for coursework, Docker, and AWS console work.
External display
Second monitor when at the desk — IDE on one screen, terminal & docs on the other.
Audio
Wired headphones for focus blocks. No noise-cancelling theatre.
Workspace
Melbourne. Most weeks: a quiet corner at home or an RMIT study space.

Editor & development

Editor
Visual Studio Code with the usual extensions: ESLint, Prettier, GitLens, Live Server, Docker. Dark theme.
Terminal
Windows Terminal with WSL2 (Ubuntu) for any Linux work; PowerShell for native Windows. SSH for everything remote.
Version control
Git on the command line; GitHub for hosting. Branch-per-feature, conventional commits, PRs even when I'm the only reviewer.
Browser
Firefox for browsing; Chrome / Chromium for DevTools-heavy debugging. Vivaldi for personal stuff.
API testing
Postman for REST API exploration. curl when I'm already in the terminal.

Languages & runtimes

Daily
JavaScript (ES6+), Bash, SQL. Whatever the task needs.
Comfortable
Python, Java, PHP. Used in different projects throughout the degree.
Have shipped
Node.js, React, Express, Wix Velo. PHP + MySQL stacks for older work.
Curious about
Rust (currently on the LinkedIn Learning shelf), Go (next).

Cloud & infrastructure

Cloud
AWS — EC2, VPC, S3, IAM, RDS, Lambda, CloudWatch, EBS/EFS, ELB & Auto Scaling. Hands-on from Cloud Foundations and DevOps coursework.
Containers
Docker for everything reproducible. Compose for local multi-service setups.
Infrastructure-as-Code
Terraform for provisioning, Ansible for configuration management.
CI/CD
GitHub Actions — builds, tests, PR checks. Branch protections on main.
Linux distro
Ubuntu Server (LTS) on EC2. vim on remote hosts because that's what's installed.

Networking & support tooling

Network analysis
Cisco Packet Tracer for design and simulation; Wireshark when I need to actually see what's on the wire.
Diagnostics
ping, traceroute, nslookup, dig, ipconfig/ifconfig, netstat, ss. The boring fundamentals.
Linux firewalling
iptables (from the Cyber Security HD assignment) — rule chains, NAT, basic hardening.

Communication & productivity

Writing
Markdown for everything I can. Notion for project notes; Apple Notes / OneNote for quick capture.
Office
Microsoft Office Suite when collaborating with non-technical clients. Excel for stock-take.
Design
Figma for wireframes and design tokens — used heavily on the Share & Swap Shop client project.
Email
Gmail. Aggressive filters.

Studying
